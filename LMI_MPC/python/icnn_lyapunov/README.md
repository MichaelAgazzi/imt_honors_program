# Gamma ICNN Lyapunov RMPC

This folder contains a PyTorch implementation for learning a lower
approximation of the CLQR/RMPC value `gamma*(x)` and using it online as a
Lyapunov-like descent objective.

## Files

- `models.py`: `GammaICNN`, an Amos-style ICNN. All `W_z` blocks are projected
  with `clamp_(min=0)` after optimizer steps to preserve convexity.
- `data.py`: CSV parser for the existing `x*_init{i}` / `gamma_{i}` simulation
  layout.
- `train_gamma_icnn.py`: training script with MSE plus over-estimation penalty
  `relu(gamma_hat - gamma_true)^2`.
- `controller.py`: `compute_control_action`, which optimizes `u` by autograd
  through `gamma_hat(Ax + Bu)` and projects onto actuator saturation bounds.

## Training

The default dataset is `simulation_data_gamma.csv`, which already contains the
`gamma_*` columns generated by the SDP solver. The file
`simulation_data_bigger_var_ep01.csv` has no `gamma_*` target columns, so it
cannot train this value-function ICNN unless regenerated with gamma.

```powershell
python .\icnn_lyapunov\train_gamma_icnn.py --calibrate
```

## Control Example

```python
import torch
from icnn_lyapunov import GammaICNN, compute_control_action

ckpt = torch.load("icnn_lyapunov/gamma_icnn.pth", map_location="cpu")
model = GammaICNN(
    input_dim=ckpt["input_dim"],
    hidden_sizes=ckpt["hidden_sizes"],
    activation=ckpt["activation"],
    enforce_positive_output=ckpt["enforce_positive_output"],
)
model.load_state_dict(ckpt["model_state_dict"])

result = compute_control_action([0.5, 0.2], model, u_max=2.0)
print(result.u, result.stable, result.lyapunov_delta)
```

## Closed-Loop Simulation

Use the SciPy-based closed-loop simulator when you want to replace the online
`lmi_clqr` call with the trained `GammaICNN` controller. The default controller
uses a short prediction horizon and a running state cost, because for this motor
the input does not affect `x1` directly in one step.

```powershell
python .\icnn_lyapunov\simulate_closed_loop.py --checkpoint icnn_lyapunov\gamma_icnn.pth --nsim 30 --u-max 2.0
```

## Theta-Based Control

The main controller can optimize `theta = (Q, Y)` instead of optimizing `u`
directly. It parametrizes `Q = L L^T + eps I`, computes `F = Y Q^{-1}`, applies
`u = F x_k`, and penalizes violations of
`gamma_{k+1} - gamma_k < -B` with `B = beta ||x_k||^2`.

```powershell
python .\icnn_lyapunov\simulate_closed_loop.py --control-mode theta --nsim 80 --beta 1e-6 --state-weight 10.0 --vertex-id 0
```

The current CSV files do not contain `Q` and `Y`, so the checkpoint trained from
`simulation_data_gamma.csv` is a `gamma(x)` model. The theta controller still
works by optimizing `theta` through `x_{k+1}=Ax_k+Bu_k`. For a true
`gamma(x,theta)` ICNN, regenerate the dataset saving `Q_val` and `Y_val`, then
train with features `[x1, x2, q11, q12, q22, y1, y2]`.

Generate a compact theta dataset:

```powershell
python .\icnn_lyapunov\generate_theta_dataset.py --n-x0 8 --nsim 25
```

Train a true `gamma(x, theta)` ICNN:

```powershell
python .\icnn_lyapunov\train_gamma_icnn.py --dataset icnn_lyapunov\simulation_data_theta_small.csv --feature-mode theta --output icnn_lyapunov\gamma_icnn_theta.pth --calibrate
```

Then run the theta controller with that checkpoint:

```powershell
python .\icnn_lyapunov\simulate_closed_loop.py --checkpoint icnn_lyapunov\gamma_icnn_theta.pth --control-mode theta --nsim 80 --beta 1e-6
```

The simulator automatically supports two checkpoint types:

- `feature_mode = state_u`: the network is interpreted as `gamma(x, u)` and SciPy
  minimizes over `[x1, x2, u]`.
- `feature_mode = state`: the network is interpreted as `gamma(x)` and SciPy minimizes
  `gamma(x_next)` with `x_next = motor_step(x, u)`.

## Learned Actor With Robust Lyapunov Check

`train_value_actor.py` trains two networks:

- `ICNNLyapunov`: `V_hat(x) = phi(x) - phi(0) + eps ||x||^2`, with ICNN
  hidden-to-hidden weights projected nonnegative.
- `ActorNetwork`: a supervised actor `u = pi_theta(x)` trained from the CLQR
  control column.

The script uses `gamma` as `V_star` when no explicit `V_star`, `value`, or
`cost_to_go` column exists. It estimates empirical and Lipschitz-corrected
`beta_minus`/`beta_plus`, then verifies

```text
V_hat(f(x, pi_theta(x))) - V_hat(x) + beta_minus + beta_plus + c_alpha ||x||^2 <= 0
```

or the symmetric `2 beta` version with `--use-symmetric-beta`.

```powershell
python .\icnn_lyapunov\train_value_actor.py --dataset icnn_lyapunov\simulation_data_theta_small.csv --pretrain-epochs 180 --epochs 30 --lambda-decrease 0.05 --plot-nsim 80
```

Outputs include a checkpoint, a printed beta/stability report, and plots for
training history, value fit, approximation error, certified beta tube,
Lyapunov decrease, robust violation, state-space diagnostics, actor fit,
closed-loop trajectories, state norm decay, and control input.

For a conservative post-training certificate estimate, rerun with
`--use-lipschitz-correction`. This usually makes the robust condition much more
conservative; it is useful for reporting, but it can be too strict to use as a
training penalty on this small dataset.
